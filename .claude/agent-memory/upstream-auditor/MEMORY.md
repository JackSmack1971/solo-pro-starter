# Upstream Auditor Memory

## Persistent Priorities

- Confirm evidence before issue creation.
- Prefer one focused issue per real problem over broad omnibus findings.
- Keep downstream fix scope small and implementation-ready.

## Known Risk Surfaces

- dependency and supply-chain drift
- wallet, chain, ABI, artifact, or release inconsistencies that span multiple files
- duplicate findings or weak findings that would create noisy issues

## Handoff Checklist

- Each finding is evidence-backed
- Impact is explicit
- Fix scope is isolated
- Verification steps are concrete
